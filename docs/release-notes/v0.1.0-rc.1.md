# Atlas UI v0.1.0-rc.1 Release Notes

Status: Draft
Drafted: June 20, 2026

## Summary

`v0.1.0-rc.1` is a release-candidate hardening cut for Atlas UI. This draft records the current package surface, captures the intentional non-export decisions from the legacy migration audit, and keeps the `atlas-ui` package boundary fixed while RC work is in progress.

## Public Package Surface

- Package name remains `atlas-ui`.
- Public exports remain limited to `src/primitives`, `src/patterns`, `src/types`, and `src/headless`.
- `docs/public-api-manifest.md` currently records `59` shipped public components: `29` primitives and `30` patterns.
- `examples/` remains Storybook-only and documentation-only reference material.

## Audit and Quarantine Decisions

- `docs/migration-map.md` tracks `78` legacy audit rows.
- `58` audit rows are marked as public-export work and are already built.
- `20` audit rows are intentionally non-export: `2` `REFERENCE_ONLY`, `2` `MERGED`, and `16` `EXAMPLE_ONLY`.
- `docs/not-built.md` is the RC source of truth for why those `20` rows do not become standalone public exports.

## Included in This RC Documentation Pass

- Added `docs/not-built.md` to document non-export migration decisions.
- Added this draft release note under `docs/release-notes/v0.1.0-rc.1.md`.
- Preserved the existing Atlas UI rule that the component library ships capabilities while apps ship meaning.

## Known Bookkeeping Note

`Select` is a shipped public primitive with Storybook and test coverage, and it is counted in `docs/public-api-manifest.md`. It does not currently have its own dedicated row in `docs/migration-map.md`; the current manifest treats it as a shipped implementation of the `Select shell` baseline from `docs/component-disposition.md`.

This is a documentation bookkeeping nuance, not a reason to widen RC scope or add a new public surface area during hardening.

## Verification

Release-note drafting for `v0.1.0-rc.1` is complete when:

- the package surface stays within the current Atlas UI export boundary,
- the non-export audit rows remain quarantined as documented in `docs/not-built.md`,
- and `npm run check` passes on the RC branch.
